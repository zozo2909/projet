Les formulaires

Un formulaire est créé avec <form>. Ses attributs définissent comment les données sont envoyées.

<form action="traitement.php" method="POST">
  <!-- champs -->
  <button type="submit">Envoyer</button>
</form>
  • action — URL qui reçoit les données.
  • method="GET" — Données visibles dans l'URL (recherches).
  • method="POST" — Données cachées, pour les infos sensibles.

Exemple de formulaire de contact

<form action="contact.php" method="POST">

  <label for="nom">Nom :</label>
  <input type="text" id="nom" name="nom" required>

  <label for="email">Email :</label>
  <input type="email" id="email" name="email" required>

  <label for="message">Message :</label>
  <textarea id="message" name="message" rows="5"></textarea>

  <input type="checkbox" id="cgu" name="cgu" required>
  <label for="cgu">J'accepte les conditions</label>

  <button type="submit">Envoyer</button>
</form>

Attributs importants des champs

  • name — Identifiant côté serveur. Indispensable pour la transmission des données.
  • id — Pour lier le champ à son <label> via for.
  • placeholder — Texte indicatif quand le champ est vide.
  • required — Champ obligatoire, le navigateur bloque l'envoi si vide.
  • disabled — Désactive le champ.
  • maxlength — Nombre maximum de caractères.

Bonnes pratiques

  • Toujours utiliser POST pour les données sensibles.
  • Toujours associer chaque <label> à son champ.
  • Ne jamais faire confiance à la validation HTML seule — valider côté serveur.
