Les images

La balise <img> affiche une image. C'est une balise auto-fermante avec deux attributs essentiels.

<img src="chemin/image.jpg" alt="Description de l'image">
  • src — Chemin vers le fichier image (local ou URL web).
  • alt — Texte alternatif si l'image ne charge pas. Indispensable pour l'accessibilité.

Les attributs de <img>

AttributExempleDescription
srcsrc="photo.jpg"Chemin de l'image — obligatoire
altalt="Un chien"Texte alternatif — obligatoire
widthwidth="400"Largeur en pixels
heightheight="300"Hauteur en pixels
loadingloading="lazy"Chargement différé (performance)
titletitle="Info-bulle"Info-bulle au survol
<!-- Image locale avec dimensions -->
<img src="images/chat.jpg"
     alt="Un chat roux sur un canapé"
     width="600" height="400">

<!-- Image avec chargement différé -->
<img src="photo.jpg" alt="Paysage" loading="lazy">

<!-- Image décorative (alt vide = ignorée par lecteurs d'écran) -->
<img src="deco.png" alt="">

Les formats d'image

  • .jpg — Photos. Bonne compression, légère perte de qualité.
  • .png — Images avec transparence. Qualité parfaite, fichiers plus lourds.
  • .svg — Icônes vectorielles. Infiniment redimensionnable sans perte.
  • .webp — Format moderne. Meilleure compression que jpg/png.
  • .gif — Images animées simples. Limité à 256 couleurs.

</a>

Figure avec légende

<figure>
  <img src="mont-fuji.jpg" alt="Le Mont Fuji">
  <figcaption>Le Mont Fuji, Japon</figcaption>
</figure>

Bonnes pratiques

  • Toujours renseigner alt — description précise pour les images informatives, alt="" pour les décorations.
  • Spécifier width et height pour éviter les sauts de page au chargement.
  • Utiliser loading="lazy" pour les images en bas de page.
  • Optimiser le poids des images — viser moins de 200 Ko.
