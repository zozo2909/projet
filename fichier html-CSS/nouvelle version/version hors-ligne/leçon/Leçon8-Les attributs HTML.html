Les attributs HTML

Un attribut se place dans la balise ouvrante et donne des informations supplémentaires à l'élément.

<balise nom="valeur">Contenu</balise>

<!-- Exemple -->
<a href="https://exemple.com" target="_blank">Lien</a>

<!-- Attributs booléens (sans valeur) -->
<input type="text" required disabled>

Les attributs universels

AttributRôleExemple
idIdentifiant unique sur la pageid="titre-principal"
classClasse CSS, réutilisable sur plusieurs élémentsclass="card active"
styleCSS en ligne directement sur l'élémentstyle="color:red"
titleInfo-bulle au survoltitle="En savoir plus"
langLangue du contenulang="en"
hiddenCache l'élément (booléen)hidden
data-*Données personnaliséesdata-id="42"

id vs class

<!-- id : unique, une seule fois par page -->
<h1 id="titre-principal">Mon site</h1>

<!-- class : réutilisable sur plusieurs éléments -->
<div class="carte">Carte 1</div>
<div class="carte">Carte 2</div>

<!-- Plusieurs classes sur un élément -->
<button class="btn btn-primaire grand">Valider</button>

Les attributs data-*

Permettent de stocker des données personnalisées dans le HTML, lisibles par JavaScript.

<button data-action="supprimer" data-id="42">Supprimer</button>

// Lecture en JavaScript :
// element.dataset.id      → "42"
// element.dataset.action  → "supprimer"

Les attributs ARIA

Améliorent l'accessibilité pour les lecteurs d'écran.

  • aria-label — Nom accessible pour un élément sans texte visible.
  • aria-hidden="true" — Cache l'élément aux technologies d'assistance.
  • aria-expanded — Indique si un menu est ouvert ou fermé.
<!-- Bouton avec seulement une icône -->
<button aria-label="Fermer la fenêtre"></button>

Bonnes pratiques

  • id est unique — ne jamais l'utiliser deux fois.
  • Éviter le style en ligne — préférer les classes CSS.
  • Nommer les attributs de façon descriptive.
  • Toujours mettre les valeurs entre guillemets.
